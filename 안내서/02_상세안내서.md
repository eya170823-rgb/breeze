## 1. 전체 구조

```
[구글시트 매물장] ──┐
                    ├─→ [앱스 스크립트 엔진] ──→ [홈페이지] ──→ 손님
[네이버 블로그]   ──┘     (개인정보 제외,         (자동 표시)
                          매물/뉴스/게시판 가공)
```

- **데이터 출처**: 매물 = 구글시트 / 뉴스·게시판 = 네이버 블로그
- **엔진**: 구글 앱스 스크립트(별개 프로젝트)가 시트·블로그를 읽어 **안전한 정보만** 홈페이지로 전달
- **홈페이지**: 순수 HTML/CSS/JS (서버 불필요, GitHub Pages로 무료 호스팅)
- **연동 방식**: 손님이 페이지를 열 때마다 최신 데이터를 읽음 → **새로고침하면 실시간 반영**

## 2. 디자인 — 색·서체

### 색상 팔레트
| 용도 | 색 | 코드 |
|---|---|---|
| 배경(메인) | 퓨어 화이트 | `#ffffff` |
| 보조 면 | 연한 회색 | `#f6f5f3` |
| 포인트(강조) | **뮤트 테라코타** | `#bd6a52` |
| 포인트(진한) | 진한 테라코타 | `#a3563f` |
| 본문 글자 | 진한 먹색 | `#1c1a19` |
| 보조 글자 | 회색 | `#57534f` / `#8c877f` |
| 어두운 섹션 | 블랙 | `#131110` / `#1c1a19` |

> 색을 바꾸려면 `styles.css` 맨 위 `:root` 의 `--terra` 등 값을 수정. (한 곳만 바꾸면 전체 반영)

### 서체
- **Pretendard** (깔끔한 현대 고딕) — 본문·제목 전부 통일
- CDN으로 로딩, `styles.css`의 `--font-display` 한 줄로 전체 제목 폰트 제어
- 한글 단어가 줄 끝에서 쪼개지지 않게 `word-break: keep-all` 적용

### 로고·아이콘
- 헤더/푸터 로고: `assets/logo.png` (투명배경 처리됨)
- 브라우저 탭 아이콘(파비콘): `assets/favicon.png` (테라코타 + 흰 로고)
- 슬로건: **"바른 기준, 가치 있는 동행 / BREEZE가 당신과 함께합니다"**

## 3. 페이지별 설명

### 홈 (`index.html`)
- **히어로**: 슬로건 + 특징 3종(사람 중심 / 1:1 밀착중개 / 신뢰 중개, 아이콘)
- **소개**(검정 배경): 회사 소개 + 취급분야(매매·임대 / 상가·토지 / 맞춤 상담)
- **한눈에 보기**: 매물·뉴스·게시판 바로가기 카드
- **매물 접수 안내**: "매물을 내놓으시나요?"
- **상담 CTA**: 전화·문자(·카카오) 아이콘
- **푸터**: 오시는 길(주소·전화·이메일·영업시간·주차) + 네이버지도/카카오내비 + 채널 아이콘 + 사업자정보

### 매물 (`매물.html`)
- 상단 **카테고리 아이콘 4종**(아파트·원투룸·오피스텔·상가임대) → 누르면 지도+지역별 매물 수
- **필터**: 종류 / 지역(동) / 거래 / 가격 (당근식 알약 → 바텀시트)
- **검색창**: 지역·건물명·키워드
- **매물 리스트**: 가로형 카드 (왼쪽 사진 / 오른쪽: 제목·가격·종류/평/층·매물번호)
- 카드 클릭 → **상세 팝업**(정보 + 지도 + 전화문의)

### 뉴스 (`뉴스.html`) / 게시판 (`게시판.html`)
- 네이버 블로그 카테고리 글을 카드로 표시 (제목·요약·날짜·블로그 링크)

## 4. 매물 시스템 (핵심)

### 공개 기준
- 구글시트 매물장의 **`등록자` 칸 = `광고완료`** 인 매물만 홈페이지에 노출
- (앞으로 오일장·직방·당근 등 광고 채널을 통합 관리하는 개념의 "광고완료" 표시)

### 상태 처리
| 상태 (`구분` 또는 `상태` 칸) | 홈페이지 |
|---|---|
| 공실 | 정상 표시 |
| 임대중 · 계약완료 | 사진 **회색** + 가운데 **상태 글자**, 목록 맨 뒤로 |
| 거래완료 | **자동 숨김(소멸)** |

### 종류별 제목 규칙
| 종류 | 제목 | 예 |
|---|---|---|
| 아파트 | 동 + 아파트명 | 일도이동 삼화아파트 |
| 원투룸 | 동 + 타입 | 노형동 원룸 |
| 오피스텔 | 오피스텔명(+타입) | 연무정길40 |
| 상가 | **업종** (상호는 숨김) | 식당 / 미용실 |
| 그 외 | 건물명/지역 + 타입 | |

### 상가·토지 등 특별 처리
- **상가·상가건물·토지·공장창고·기타매물**: **주소·지도 숨김** (정확한 위치 비공개)
- **상가·상가건물**: **권리금 금액 숨김** → "권리금 협의"로 표시
- **상가**: **상호(가게 이름) 숨기고 업종**으로 표시
- 단, 지역(동) 단위 집계는 가능 (지역 필터·지도용)

### 가격 표기
- 숫자(만원)면 자동 변환: `40900` → "4억 900만원"
- 월세/연세: "보증 5,000 / 월 350" 형식

### 사진·워터마크
- 사진 칸에 **웹 이미지 주소(https://...jpg)** 가 있으면 표시 (앱시트 드라이브 폴더 링크는 표시 안 됨)
- 사진 위에 **BREEZE 워터마크**(대각선 은은한 반복) 자동 적용
- **우클릭·드래그 저장 차단** (퍼가기 방지 — 완벽 차단은 아니나 도용 표시 효과)
- 사진 없으면 **회색 BREEZE 기본 썸네일**

## 5. 뉴스·게시판 (블로그 연동)
- 네이버 블로그 RSS를 엔진이 서버에서 읽어옴 (안정적)
- **부동산뉴스** 카테고리 → 뉴스 / **게시판** 카테고리 → 게시판
- ⚠️ 두 카테고리는 블로그 **상위(독립) 카테고리**로 둬야 RSS에 인식됨 (하위 카테고리는 안 잡힘)
- 글 올리면 자동 표시, 클릭 시 블로그 글로 연결

## 6. 자동화 엔진 (앱스 스크립트)
- 파일: `apps-script/홈페이지엔진.gs`
- **별개(standalone) 프로젝트** — 기존 앱시트/캘린더 코드와 분리되어 서로 안 건드림
- 시트는 `SHEET_ID`로 연결 (시트 비공개 가능)
- 엔드포인트: `?action=listings`(매물) / `?action=news` / `?action=board`
- **자동 갱신**: `installDailyTriggers()` 한 번 실행 → 매일 아침 7시 광고완료 매물의 **등록일을 오늘로** 자동 갱신 (신선도 유지)
- 홈페이지 연결: `config.js`의 `feedUrl`에 웹앱 주소 입력 (설치법: [엔진-설치가이드.md](../엔진-설치가이드.md))

## 7. 들어간 디테일 모음
대화하며 반영한 디테일들 (시간순 요약):
- 모바일 우선 반응형 / 인터랙티브(스크롤 애니메이션·햄버거 메뉴)
- 색: 퓨어 화이트 + 그레이 + 뮤트 테라코타 / 로고·파비콘 적용
- 슬로건·소개 문구, 특징을 **사람 중심**으로 강조
- 상담 버튼을 **아이콘(전화·문자·카카오)** 으로
- 푸터: 오시는 길 + **네이버 지도 / 카카오내비** 길안내, 채널 아이콘(네이버·카톡·유튜브·인스타·블로그·전자책), 사업자정보(상호·대표자 박정희·사업자번호)
- 이메일·영업시간(09:00–20:00)·주차(전면주차) 추가
- 매물: **당근식 필터**(종류·지역·거래·가격, 바텀시트, 다중선택), 카테고리 아이콘+**지역 지도**(매물 수 버블)
- 가로형 리스트 카드 / PC 2열 / 상세 팝업에 **지도** 연결
- 종류별 제목 규칙, 상태(회색/숨김), 상가 업종·권리금·주소 처리
- **사진 워터마크 + 퍼가기 차단**
- PC에선 팝업 **가운데 모달**, 모바일은 바텀시트
- 한글 줄바꿈(keep-all), 필터 줄바꿈, 푸터 정렬·비중 축소
- 캐시 문제 해결용 no-cache 테스트 서버
- **앱스 스크립트 엔진** + 매일 자동 갱신 (개인정보 완전 제외 검증 완료)

## 8. 수정 방법
- **간단한 글자**(전화·주소·영업시간·사업자번호·이메일·주차·슬로건): `config.js`의 따옴표 안 글자 수정
- **색·여백·크기**: `styles.css` (특히 `:root` 변수)
- **블로그 카테고리·시트 ID·엔진 주소**: `config.js`
- 인터넷 공개 후: github.com에서 파일 ✏️ 편집 → 커밋 → 1~2분 후 반영
- 매물·뉴스 내용: 시트·블로그에서 (코드 수정 불필요)

## 9. 주의사항
- ⚠️ 구글시트 **탭/열 이름 변경 금지** (앱시트·엔진 깨짐)
- ⚠️ **공개 사이트(config.js)에 `sheetId` 넣지 말 것** — 개인정보 노출. 반드시 `feedUrl`(엔진) 사용
- ⚠️ 엔진 코드 수정 후엔 **배포 → 새 버전**으로 다시 배포해야 반영됨
- 블로그 카테고리는 **상위 카테고리**로
- 사진은 **웹 주소**여야 표시됨 (드라이브 폴더 링크 ✕)

## 10. 파일 구조
```
홈페이지만들기/
├─ index.html / 매물.html / 뉴스.html / 게시판.html   ← 페이지
├─ styles.css        ← 디자인(색·서체·레이아웃)
├─ config.js         ← ★설정(전화·주소·사업자번호·엔진주소 등)
├─ layout.js         ← 공통 헤더/푸터
├─ app.js            ← 화면 동작(필터·카드·팝업·지도·워터마크)
├─ sheets.js         ← 데이터 불러오기(엔진/블로그)
├─ sample-data.js    ← 샘플(연결 전 표시용)
├─ assets/           ← 로고·파비콘
├─ apps-script/홈페이지엔진.gs   ← ★엔진(구글 앱스 스크립트에 설치)
└─ 안내서/           ← 이 문서들
```
